1. Принимает номер задачи (например, "123-описание" или просто номер) (plans/tasks.md)
2. Создаёт новую ветку Git с именем `feature/sub/<номер задачи>-<описание>` (если описание не предоставлено, использует только номер)
3. Реализует требования задачи (вносит изменения в код)(plans/Добавление телеграм бота для напоминаний о работе.md)
4. Напиши тесты
5. Запусти, добейся, того что они проходят
6. Создаёт коммит с сообщением, содержащим номер задачи (формат: "Task #<номер>: <описание изменений>") 
7. Ожидай ревью, жди подтверждение, что пройдено 
8. Находит чек-лист в проекте (plans/checklist_telegram_bot.md)
9. Находит соответствующую задачу в чек-листе и заменяет `[ ]` на `[x]`
10. Создаёт второй коммит только с изменениями в чек-листе (сообщение: "Task #<номер>: отмечена в чек-листе")
